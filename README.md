# MBA Essay Writing Skill

一个为 MBA 学生设计的 AI 写作提示词，帮你用《哈佛商业评论》的风格完成课程写作。

---

## 对 MBA 学生有什么用？

**你面对的问题：**

- 工作/学业紧张，我不想在这堂课花费太多时间，但我想超过那些没有过脑子，完全用AI生成的同学；
- 拿到题目不知道从哪里下手，对着空白文档发呆；
- AI 帮你写出来的东西一眼就能看出是 AI 写的：满屏破折号、"不是……而是……"、"综上所述"
- 洋洋洒洒写了 2000 字，引用了根本查不到的数据，只好删掉重写；

**这个 Skill 怎么解决：**


| 痛点     | 解法                                             |
| ------ | ---------------------------------------------- |
| 不知道写什么 | 第一步收集信息 + 第二步追问，帮你把模糊想法变成清晰论点                  |
| 结构乱    | 第三步自动判断文体，第四步按 HBR 框架生成提纲，你确认后再动笔              |
| AI 腔调重 | 你粘贴自己的范文，AI 会模仿你的语气写，不是模板腔                     |
| 数据造假   | 第六步专门做幻觉校验：每个数据必须有出处，找不到的直接删                   |
| 引用混乱   | 自动生成 materials.md 收集搜索资料，essay.md 用 [n] 标注引用位置 |


**核心逻辑：** 不是让 AI 替你写，而是六步走完一个从"模糊想法"到"有立场、有论据、有来源"的完整写作流程。最后那篇文章，是你的观点，AI 负责结构和语言。

## 怎么用

### 写作流程（六步）

启动 Skill 后，AI 会引导你走完以下流程，每步完成后等你确认再推进：


| 步骤           | 做什么                                                         | 你需要做什么  |
| ------------ | ----------------------------------------------------------- | ------- |
| **第一步** 收集信息 | AI 问你：课程主题、题目要求、字数、初步想法                                     | 填写，可以模糊 |
| **第二步** 深度追问 | AI 针对性追问 3 个问题，挖掘你真正的论点和素材                                  | 回答问题    |
| **第三步** 判断文体 | AI 自动判断是 Case Analysis / Reflection Essay / Business Report | 确认或纠正   |
| **第四步** 生成提纲 | 按 HBR 风格生成章节结构，写入 `提纲.md`                                   | 确认或修改   |
| **第五步** 生成正文 | 按提纲写作，联网搜索佐证资料，生成 `essay.md` 和 `materials.md`               | 等待，然后审阅 |
| **第六步** 幻觉校验 | 逐一核查每条数据的出处，无出处的直接删除，有出处的标注引用 [n]                           | 最终审阅定稿  |


---

### 准备工作：加入你自己的范文（强烈建议）

打开 `skill.md`，找到文件末尾的 `文笔参考` 部分，把你自己写过的任何文章粘贴进去。

```
文笔参考（可选，但强烈建议填写）：

<!--
<在此粘贴你的范文>
-->
```

范文越真实越好，可以是：课程作业、工作汇报、公众号文章、申请文书。字数不限，500 字以上效果更明显。**这是让 AI 理解你个人写作风格的最有效方式，比任何风格描述都准确。**

### 在 OpenClaw 中使用

[OpenClaw](https://openclaw.ai) 是一个开源的自托管 AI 助手，可接入 WhatsApp、Telegram、Slack 等你日常使用的消息平台。本项目本身就是一个 OpenClaw Skill，可以直接安装使用。

**前提条件：** 需要准备一个 Claude API Key（[申请地址](https://console.anthropic.com/)）。

**安装步骤：**

1. 将本项目文件夹放入 OpenClaw 的 skills 目录（参考 [官方文档](https://docs.openclaw.ai) 获取路径）
2. 确认文件夹内有 `skill.md`（OpenClaw 会自动识别）
3. 在你绑定的消息平台中发送 `/essay-writing` 或对应的 Skill 触发词
4. 按提示填写你的 API Key，之后直接开始对话即可

> **推荐场景：** 习惯在微信/Telegram/WhatsApp 里处理事务的同学，不需要额外开任何 APP，直接在消息平台里完成整个写作流程。

### 在 Cursor 中使用

1. 在你的写作项目根目录创建 `.cursor/rules/essay-writing.mdc`
2. 将 `skill.md` 全部内容复制进去
3. 在 Cursor 的 Chat 中 `@essay-writing` 引用该规则，然后说"开始"

> **推荐场景：** 想在本地管理所有写作文件（提纲.md / essay.md / materials.md），方便版本管理。

### 在 Claude Code 中使用

```bash
# 克隆或下载本仓库
git clone <repo-url>
cd mba-essay-writing-skill

# 启动 Claude Code
claude
```

在 Claude Code 中，将 `skill.md` 配置为自定义 Skill：

1. 将 `skill.md` 放入 `~/.claude/skills/` 目录（或项目的 `.claude/skills/`）
2. 在对话中输入 `/essay-writing` 触发

或者直接在对话中粘贴 `skill.md` 内容开始使用。

> **推荐场景：** 需要 AI 自动写入文件（提纲.md、essay.md、materials.md）并管理写作过程，Claude Code 可以直接操作本地文件系统，体验最完整。

## 推荐模型


| 模型                          | 推荐程度     | 说明                       |
| --------------------------- | -------- | ------------------------ |
| **Claude Sonnet 4.5 / 4.6** | ★★★★★ 首选 | 写作质量最高，理解细腻，搜索 + 写作能力平衡好 |
| Claude Opus 4.6             | ★★★★☆    | 质量更高但速度慢                 |
| Claude Haiku 4.5            | ★★★☆☆    | 速度快、成本低，但质量略差            |
| GPT/Gemini                  | ★★★☆☆    | 可用，但中文写作风格不如 Claude 自然   |
| 国产开源模型                      | -        | 未测试，Prompt很长，经验来说效果不佳    |


> **结论：** 日常使用 Claude Sonnet，最终润色考虑 Opus。这个 Skill 是为 Claude 优化的，在其他模型上效果可能有所不同。

## 输出文件说明

运行完整流程后，会在当前目录生成：

```
提纲.md       ← 第四步生成，章节结构和叙述要点
essay.md      ← 第五步生成，正文 + 写作说明 + 参考资料
materials.md  ← 第五步生成，所有搜索到的资料来源
```

---

## 支持的文体类型

- **案例分析（Case Analysis）** — 情境定义 → 问题诊断 → 选项分析 → 决策建议 → 执行考量
- **自我职业生涯反思（Reflection Essay）** — 触发时刻 → 经历描述 → 认知转变 → 行动启示 → 核心立场
- **商业报告（Business Report）** — 核心结论 → 问题背景 → 分析框架 → 关键发现 → 行动建议 → 风险与局限

---

## 不支持的文体类型

- **财务分析（Financial Analysis）** — 数据太多，幻觉率大增；
